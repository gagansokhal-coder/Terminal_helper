# README Screenshots Plan

This document outlines the visual assets (screenshots/GIFs) needed for the README to demonstrate `ggnmem`'s core features.

### 1. TUI Search Interface
*   **Purpose:** Showcase the interactive full-screen `Ctrl+R` replacement, including the UI layout, command preview, and mode cycling.
*   **Example command:** `ggnmem ui`
*   **Recommended terminal size:** 80x24 or 100x30
*   **Suggested filename:** `tui-search.gif`

### 2. Semantic Search Example
*   **Purpose:** Demonstrate the power of intent matching by showing a search for a concept that retrieves commands with completely different keywords.
*   **Example command:** `ggnmem semantic "postgres backup"`
*   **Recommended terminal size:** 80x15
*   **Suggested filename:** `semantic-search.png`

### 3. Natural Language Ask Command
*   **Purpose:** Show the built-in knowledge base and how users can ask for commands in plain English.
*   **Example command:** `ggnmem ask "show running containers"`
*   **Recommended terminal size:** 80x12
*   **Suggested filename:** `ask-command.png`

### 4. Self-Update Command
*   **Purpose:** Highlight the frictionless, in-place upgrade experience.
*   **Example command:** `ggnmem self-update`
*   **Recommended terminal size:** 80x15
*   **Suggested filename:** `self-update.gif`

### 5. One-Line Installer
*   **Purpose:** Prove how easy the setup is by showing the bootstrap script in action.
*   **Example command:** `curl -fsSL https://raw.githubusercontent.com/gagansokhal-coder/Terminal_helper/main/scripts/install-online.sh | bash`
*   **Recommended terminal size:** 100x20
*   **Suggested filename:** `installer.gif`
